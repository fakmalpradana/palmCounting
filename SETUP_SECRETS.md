## Alternative: Service Account Key Method (Less Secure)

> ⚠️ Only use this if WIF doesn't work. SA Keys are long-lived credentials
> and should be rotated regularly. **WIF (above) is strongly preferred.**

```bash
# Generate a JSON key
gcloud iam service-accounts keys create /tmp/sa-key.json \
  --iam-account=github-actions-deploy@palmcounting.iam.gserviceaccount.com \
  --project=palmcounting

# Print it (copy the entire output)
cat /tmp/sa-key.json

# Then add to GitHub as secret:
# Name:  GCP_CREDENTIALS
# Value: <paste the entire JSON content>

# Clean up the local key file immediately
rm /tmp/sa-key.json
```

Update `deploy.yml` to replace the `auth` step with:
```yaml
- name: Auth to GCP (SA Key)
  uses: google-github-actions/auth@v2
  with:
    credentials_json: ${{ secrets.GCP_CREDENTIALS }}
```
